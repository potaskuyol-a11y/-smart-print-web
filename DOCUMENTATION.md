## 1. Обзор продукта

**Smart Print Web** — профессиональная B2B-система для создания, согласования и управления коммерческими предложениями (КП) на программное обеспечение Smart Print.

### Ключевые возможности

- Расчёт стоимости лицензий для 4 типов функционала
- Многоуровневое ценообразование (Дистрибьютор / Партнёр / РРЦ)
- Поддержка годовых и бессрочных лицензий
- Автоматический подбор оборудования по типу функционала
- Workflow согласования (Draft → In Review → Approved → Sent)
- Подключение Центра компетенций (ЦК) для подбора оборудования Тип 4
- Работы по внедрению и командировочные расходы
- Генерация КП в форматах PDF и Excel
- Интеграция с Bitrix24 CRM
- Импорт анкет клиентов из Excel

## 6. Модули приложения

### 6.1 Дашборд (`/dashboard`)

Главная страница после входа. Отображает таблицу всех расчётов с фильтрами.

**Фильтры:**
- Статус КП (Черновик / На согласовании / Согласован / Отправлен)
- Тип лицензий (Годовые / Бессрочные)
- Тип КП (Партнёрское / Прямое / Дистрибьюторское)
- Тип лицензии (Тип 1–4)
- Поиск по клиенту

**Индикаторы строки расчёта:**
- Цвет строки: обычный / **синий** (расчёт ожидает ЦК)
- Статус (бейдж)
- Сумма по типу КП
- Ссылка ведёт на `/calculator/{id}` или `/cc/{id}` в зависимости от роли

---

### 6.2 Калькулятор — создание (`/calculator`)

Форма создания нового КП.

**Карточка проекта:**
- Клиент (обязательное поле)
- Сделка в Битрикс24 (дропдаун активных сделок менеджера)
- Проект (текстовое поле, автозаполняется из сделки)
- Тип КП (Партнёрское / Прямое / Дистрибьюторское)
- Партнёр / Дистрибьютор (если выбран соответствующий тип)
- Тип лицензий (Годовые / Бессрочные)

**Раздел устройств:**
- Импорт из Excel-анкеты клиента
- Ручное добавление устройств
- ComboInput с поиском по реестру для Производителя и Модели
- Выбор функционала СУП (Тип 1–4)
- Автоматическая индикация статуса: Обследован / Требует обследования
- Автоматический подбор оборудования при выборе Тип 3 / Тип 4

**Раздел оборудования:**
- Отображается при наличии устройств Тип 3 или Тип 4
- Переключатель "Подбор через Центр компетенций" (требует привязки к сделке Bitrix24)
- Ручная коррекция количества и добавление позиций

**Блокировки без привязки к сделке Bitrix24:**
- Недоступно: отправка на согласование, ЦК, экспорт Excel и PDF

---

### 6.3 Калькулятор — просмотр/редактирование (`/calculator/[id]`)

Детальный вид КП с редактированием.

**Просмотр:**
- Карточка КП: клиент, проект, тип, статус, даты
- Таблица лицензий с ценами
- Таблица оборудования (с НДС)
- Таблица работ и командировок (заполняет ЦК)
- Итоговая сумма
- Комментарии менеджера / руководителя / ЦК

**Действия (зависят от роли и статуса):**
- **Редактировать** — Manager, Manager Sales
- **Копировать** — создаёт дубль расчёта
- **На согласование** — отправляет в статус `in_review`
- **Согласовать / Отклонить** — Manager, Admin
- **Скачать Excel** — требует активной сделки Bitrix24 со стадией `NEW` и выше
- **Скачать PDF** — требует стадии `EXECUTING` и выше

После скачивания PDF — модальное окно с предложением прикрепить файл к сделке в Bitrix24.

### 6.4 Центр компетенций (`/cc`, `/cc/[id]`)

Интерфейс для сотрудников ЦК.

**Список задач (`/cc`):**
- Только расчёты с `needs_cc = true`
- Фильтрация по статусу

**Работа с расчётом (`/cc/[id]`):**
- Просмотр всех позиций
- Добавление / редактирование **оборудования** (аппаратные терминалы)
- Добавление **работ по внедрению** (название, ед. изм., кол-во, цена)
- Добавление **командировочных расходов** (дни, проживание, суточные)
- Комментарий ЦК
- Кнопка "Передать руководителю" → изменяет статус на `in_review`

## 7. Типы лицензий

| Тип | Название | Описание |
|---|---|---|
| **Тип 1** | Мониторинг по сети | Сбор статистики печати на сетевых МФУ/принтерах по SNMP. Базовый мониторинг без идентификации пользователей. |
| **Тип 2** | Персональная статистика | Персонализированная статистика в разрезе пользователь/объём. Включает весь функционал Тип 1. |
| **Тип 3** | Аппаратный терминал «Катюша» | Безопасная печать с авторизацией через аппаратный терминал. Включает функционал Тип 1–2. |
| **Тип 4** | Программный терминал «Смарт Принт» | Встроенный терминал на совместимых МФУ. Только для устройств из реестра. |

### Автоматически подбираемое оборудование

| Тип | Оборудование |
|---|---|
| Тип 3 | СР-ТВТ-01, М3-КРН-01 |
| Тип 4 (Катюша) | CK223, 685621.013, 685621.015, CK121, 685621.011, СК3111, М3-КРН-02 |
| Тип 4 (прочие) | TWN4 |

## 8. Ценообразование

### Уровни цен

| Уровень | Описание |
|---|---|
| **Дистрибьютор** | Минимальные цены для дистрибьюторского канала |
| **Партнёр** | Цены для партнёрского канала |
| **РРЦ** | Рекомендованная розничная цена (прямые сделки) |

### Ступенчатое ценообразование

Цена зависит от суммарного количества лицензий в КП:
- Чем больше устройств — тем ниже цена за единицу
- Настраивается через поле `min_quantity` в `price_list`

### Тип лицензии

- **Годовые (annual)** — ежегодная подписка, не требует согласования
- **Бессрочные (perpetual)** — единоразовая покупка, **требует согласования руководителя**

### Техническая поддержка

Отдельная позиция: `C1-ETP-20` — Экспертная ТП уровня 2, пакет 10 часов (с НДС).
Количество пакетов задаётся вручную при создании КП.

## 9. Workflow согласования

```
Создан
  └→ draft (Черновик)
       ├→ [годовые + нет ЦК] → Сохранить → остаётся draft
       ├→ [бессрочные ИЛИ нет ЦК] → Отправить на согласование
       │     └→ in_review (На согласовании)
       │           ├→ [Manager/Admin] Согласовать → approved
       │           │     └→ sent (Отправлен) — вручную
       │           └→ [Manager/Admin] Отклонить → draft
       └→ [needs_cc = true] → Отправить в ЦК
             └→ in_review (ЦК заполняет оборудование/работы)
                   └→ Передать руководителю → остаётся in_review
                         └→ [Manager] Согласовать → approved
```

### Правила

- Бессрочные лицензии **всегда** требуют согласования
- Расчёт с `needs_cc = true` **всегда** проходит через ЦК
- После согласования статус `approved` позволяет скачать PDF
- Партнёры не могут отправлять на согласование напрямую

## 10. Интеграция с Bitrix24

### Конфигурация

Интеграция через **Incoming Webhook** облачного Bitrix24.

```env
BITRIX_WEBHOOK_URL=https://<domain>.bitrix24.ru/rest/<userId>/<token>/
```

### Стадии сделок

#### Активные стадии (в порядке прогресса)

| Индекс | ID стадии | Название |
|---|---|---|
| 0 | `NEW` | Клиент на пресейл |
| 1 | `PREPARATION` | Выявление потребности |
| 2 | `PREPAYMENT_INVOICE` | Демо / пилот |
| 3 | `EXECUTING` | Подготовка/согласование КП |
| 4 | `FINAL_INVOICE` | Согласование договора |
| 5 | `UC_0EI5CM` | Договор подписан |

#### Неактивные стадии (закрытые сделки)

`UC_10HZUC` (Акт подписан), `WON`, `LOSE`, `APOLOGY`

### Правила доступа к экспорту

| Условие | Excel | PDF |
|---|---|---|
| Нет привязанной сделки | ❌ | ❌ |
| Стадия `NEW` (индекс ≥ 0) | ✅ | ❌ |
| Стадия `EXECUTING` (индекс ≥ 3) | ✅ | ✅ |
| Неактивная сделка | ❌ | ❌ |

### Привязка расчёта к сделке

При создании нового расчёта:
1. Система загружает список активных сделок менеджера (по его email)
2. Менеджер выбирает сделку из дропдауна или оставляет "Без проекта"
3. `bitrix_deal_id` и `bitrix_deal_title` сохраняются в таблице `calculations`

### Прикрепление PDF к сделке

После скачивания PDF появляется модальное окно:
- "Прикрепить к сделке" → PDF загружается на диск Bitrix24 и привязывается к сделке
- "Нет, не нужно" → закрыть без прикрепления

### Методы Bitrix24 API

| Метод | Использование |
|---|---|
| `user.search` | Поиск пользователя по email |
| `crm.deal.list` | Список сделок пользователя |
| `crm.deal.get` | Данные одной сделки |
| `disk.folder.uploadfile` | Загрузка PDF на диск |
| `crm.deal.update` | Прикрепление файла к сделке |

## 11. Генерация PDF и Excel

### PDF (`lib/pdf.tsx`)

Генерируется с помощью `@react-pdf/renderer` на стороне клиента.

**Структура документа:**
1. Шапка: логотип, дата, номер КП
2. Реквизиты: клиент, партнёр/дистрибьютор, тип КП
3. Таблица лицензий (по типам)
4. Таблица оборудования (если есть, с НДС)
5. Работы по внедрению + командировочные расходы (если есть, заполняет ЦК)
6. Сводная таблица итогов
7. Условия и примечания

**Особенности:**
- Партнёр видит только партнёрские цены
- Оборудование и ТП всегда с НДС
- Командировки включаются в блок "Работы по внедрению"

### Excel (`exceljs`, динамический импорт)

Excel-файл генерируется на клиенте через `exceljs`.

**Структура:**
- Лист "КП" с форматированием
- Стилизованные заголовки
- Позиции по артикулам с ценами
- Итоговая строка

Импорт списка устройств в реестр `device_registry`.

**Входные данные:** `multipart/form-data` с Excel-файлом

**Входные данные:**

## 14. SQL-миграции

Выполнить в **Supabase Dashboard → SQL Editor**.

### Таблица командировок

```sql
-- Файл: migration_trips.sql
CREATE TABLE IF NOT EXISTS calculation_trips (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  calculation_id uuid NOT NULL REFERENCES calculations(id) ON DELETE CASCADE,
  days integer NOT NULL DEFAULT 1,
  accommodation numeric NOT NULL DEFAULT 0,
  rate numeric NOT NULL DEFAULT 0,
  sum numeric NOT NULL DEFAULT 0,
  created_at timestamptz DEFAULT now()
);

CREATE INDEX IF NOT EXISTS idx_calculation_trips_calc_id
  ON calculation_trips(calculation_id);

ALTER TABLE calculation_trips ENABLE ROW LEVEL SECURITY;

CREATE POLICY "All authenticated users can read trips"
  ON calculation_trips FOR SELECT TO authenticated USING (true);

CREATE POLICY "Competence center can manage trips"
  ON calculation_trips FOR ALL TO authenticated
  USING (
    EXISTS (
      SELECT 1 FROM profiles
      WHERE id = auth.uid() AND role IN ('competence_center', 'admin')
    )
  );
```

### Поля Bitrix24 в расчётах

```sql
ALTER TABLE calculations
  ADD COLUMN IF NOT EXISTS bitrix_deal_id   TEXT,
  ADD COLUMN IF NOT EXISTS bitrix_deal_title TEXT;
```
